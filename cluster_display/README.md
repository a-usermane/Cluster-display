# 聚类算法可视化网页

## 1. 这个项目是做什么的？

这个项目是一个面向算法讲解场景的可视化网页工具，核心目标是：

- 把抽象的聚类过程转成可观察、可操作、可解释的图形交互。
- 帮助不了解算法的人快速建立直观认知，减少“只看公式看不懂”的问题。
- 支持在讲解时实时改参数、看结果、看过程、看单点归类逻辑。

### 项目背景与灵感

在和不懂算法的人沟通时，常见问题是：

- 算法流程难以讲清楚（尤其是“为什么这个点被分到这一类”）。
- 只看结论容易产生误解（把聚类当作固定规则，而不是参数驱动过程）。
- 缺少动态演示时，听众很难把“公式”和“结果”关联起来。

因此本项目希望通过“参数输入 + 图形展示 + 公式解释 + 动画回放”的组合方式，让用户在短时间内理解算法的运行效果和运行逻辑。

## 2. 这个项目有哪些功能？

### 数据与画布

- 在中央坐标系（Canvas）中展示数据点。
- 支持按参数生成高斯分布数据集（多类、每类样本数、标准差、中心分散度、随机种子）。

### 算法支持

- 支持 `K-Means`、`DBSCAN`、`Agglomerative (Average Linkage)` 三种聚类算法。
- 聚类前统一颜色显示；聚类后按簇着色（DBSCAN 噪声点使用灰色）。

### 讲解与交互

- 参数区按算法动态显示，只呈现当前算法相关参数。
- 每个参数带有简短说明，便于快速理解作用。
- 公式区按“输入 / 计算过程 / 输出”结构化展示。
- 点击任意点可查看该点的归类解释与局部对比。
- 显示质心/簇中心标记和图例信息。

### 动画与辅助操作

- 支持算法过程动画：播放、暂停、单步、速度调节。
- 支持“一键重置”恢复默认配置。
- 支持“随机一组参数”快速生成演示场景。

## 3. 这个项目需要什么环境运行？

### 必需环境

- 现代浏览器（推荐 Chrome、Edge、Firefox 的较新版本）。

### 可选环境（推荐）

- `Node.js`（用于启动本地静态服务，如 `npx serve`）。
- 操作系统不限（Windows/macOS/Linux 均可）。

> 说明：项目是纯前端静态页面，无需数据库或后端服务。

## 4. 这个项目如何运行？

### 方式 A：直接打开（最快）

1. 进入项目目录。
2. 双击 `index.html` 用浏览器打开。

### 方式 B：本地服务运行（推荐）

1. 在项目目录打开终端。
2. 执行：`npx serve`
3. 在浏览器中访问终端输出的本地地址。

## 快速使用流程

1. 调整数据参数，点击“生成高斯数据”。
2. 选择算法并设置参数，点击“执行聚类”。
3. 点击“播放动画”查看算法过程（可暂停/单步）。
4. 点击图中任意点，查看该点的输入、计算过程和输出。

## 项目结构（简要）

- `index.html`：页面结构与控件布局
- `styles.css`：样式与可视化展示样式
- `app.js`：状态管理、渲染、事件与动画控制
- `formula.js`：公式区与单点解释渲染
- `algorithms/kmeans.js`：K-Means 实现
- `algorithms/dbscan.js`：DBSCAN 实现
- `algorithms/agglomerative.js`：Agglomerative 实现
